25/**
26 * KeyManager interface.
27 * KeyManager is a single per-application object that serves as a high-level
28 * resolver for the keys using various KeyProviders and implements the
29 * application policy in regards to:
30 * <ul>
31 * <li>key provider preference
32 * <li>key preference (i.e. which key, if any, should be preferred on
33 * collisions,
34 * <li>key lifetime (how the application should treat the old keys),
35 * <li>error handling (e.g. some provider errors can be resolved in
36 * KeyManager).
37 * </ul>
38 * @interface
39 */
40e2e.openpgp.KeyManager = function() {};

202/**
203 * Marks the key(s) as trusted for a given email address. These keys will be
204 * then returned in getKeys() calls for that user and the selected purpose(s).
205 * Other keys for that email address are implicitly marked as not trusted.
206 * The policy for trusting the keys is implemented in the KeyManager of the
207 * application.
208 * Calling this method might cause UI interaction that lets the KeyProvider
209 * manage the trusting process for the keys (e.g. entering a PIN code,
210 * confirming an action).
211 * The application may optionally pass additional data consumed by the
212 * KeyManager / KeyProvider that contain parameters for trusting keys.
213 * @param {!e2e.openpgp.Keys} keys The keys to trust.
214 * @param {!e2e.openpgp.UserEmail} email The Email address.
215 * @param {!e2e.openpgp.KeyPurposeType} purpose The purpose for which
216 * to trust the keys. Invalid purpose for a given key will be ignored.
217 * @param {e2e.openpgp.KeyTrustData=} opt_trustData Extra key trust data
218 * containing information for the KeyManager/KeyProviders.
219 * @return {!e2e.openpgp.KeysPromise} The keys.
220 * @export
221 */
